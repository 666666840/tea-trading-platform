# 前端与接口统一体验建议

## 1. 接口统一错误返回格式
- 后端所有API错误返回统一JSON格式：
  ```json
  { "error": "错误类型", "code": 400, "message": "详细描述" }
  ```
- 前端统一处理 error 字段，弹窗或页面提示。

## 2. loading/loading mask 机制
- 发起接口请求时，页面/按钮显示 loading 效果，防止重复提交。
- 请求完成后自动关闭 loading。

## 3. 全局异常捕获
- 前端全局拦截接口异常，统一弹窗提示。
- 建议使用 wx.showToast、Modal 或自定义组件。

## 4. 数据导出体验
- 导出时显示进度/提示，导出完成后自动下载。

## 5. 其它建议
- 表单提交前校验，错误即时提示。
- 重要操作二次确认。
- 统一空数据、加载中、错误页面样式。
